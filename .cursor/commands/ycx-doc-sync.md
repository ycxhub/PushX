# /ycx-doc-sync — Document Release (Technical Writer)

Sync all project documentation with recently shipped code changes. Catch stale READMEs automatically.

## Step 1: Detect Base Branch and Diff

```bash
_BASE=$(gh pr view --json baseRefName -q .baseRefName 2>/dev/null || gh repo view --json defaultBranchRef -q .defaultBranchRef.name 2>/dev/null || echo "main")
_BRANCH=$(git branch --show-current 2>/dev/null || echo "unknown")
echo "Branch: $_BRANCH | Base: $_BASE"
git diff origin/$_BASE --stat
```

## Step 2: Inventory All Documentation

Find every markdown file in the repo:
```bash
find . -name '*.md' -not -path './node_modules/*' -not -path './.git/*' -not -path './vendor/*' | sort
```

Read each documentation file. Key files to check:
- README.md
- ARCHITECTURE.md
- CONTRIBUTING.md
- CHANGELOG.md
- TODOS.md
- docs/**/*.md
- Any other .md files in the root

## Step 3: Cross-Reference Diff Against Docs

For each doc file, check if the code changes affect what the doc describes:

1. **Features described in README** — are they still accurate after the diff?
2. **Architecture descriptions** — do component diagrams match the new code?
3. **API documentation** — do endpoints, params, responses match?
4. **Setup instructions** — do install steps still work?
5. **Configuration docs** — are env vars, flags, settings current?
6. **Task/TODO files** — are completed items marked?

## Step 4: Update Stale Docs

For each stale doc:
1. Read the full doc
2. Read the relevant changed code
3. Update the doc to reflect the current state
4. Preserve the doc's existing style, tone, and structure

**Rules:**
- Never delete content that's still accurate
- Add, don't rewrite, unless the section is clearly wrong
- Preserve existing formatting and heading hierarchy
- If unsure whether something changed, leave it alone

## Step 5: TODOS.md Maintenance

If TODOS.md exists:
1. Cross-reference completed items against the diff
2. Move completed items to a `## Completed` section
3. Add completion date and version if known

## Step 6: Commit

If any docs were updated:
```bash
git add -A *.md docs/ TODOS.md
git commit -m "docs: sync documentation with shipped changes"
```

If no docs needed updating: "Documentation is current — no updates needed."

## Step 7: Report

```
DOC SYNC REPORT
═══════════════════════════════
Files scanned: N
Files updated: M
  - README.md: [what changed]
  - docs/api.md: [what changed]
Files current: K (no changes needed)
TODOS completed: L items marked done
Status: DONE
═══════════════════════════════
```

## Important Rules

- Never fabricate documentation — only update based on actual code changes
- If a doc references features you can't verify, leave it alone
- README updates should be user-facing improvements, not implementation details
- CHANGELOG is auto-generated by /ycx-ship — don't duplicate here
